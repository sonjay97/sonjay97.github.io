How I built A Redis LRU Cache With Python

Published: April 15, 2025

I recently had a project I needed to optimize the latency of calls being made to a microservice. The "microservice" in question was really just a dockerized rest API that updated prices on orders according to given rules such as the country of origin for the order (think tariff rates). The rules were stored in a Snowflake table and when order details were passed to the API (microservice, whatever you wanna call it), rules would be checked via the table. My goal was to cache the most recently used rules and get rid of all the "least recently used" ones, hence the idea for a LRU cache. But then I got to thinking, what better way to efficiently cache objects then with in-memory storage. Thus began my idea to use Redis. Now I have never used Redis before. As a somewhat new-ish developer (2 yoe as of time of writing) I didn't know what Redis was till a very senior developer at work mentioned it and piqued my curiosity. Once I did some research and realized how powerful it is I figured it would pay dividends to create a project using Redis. Redis is pretty cool. But learning how to use it to have impact on real world issues would be even cooler.
